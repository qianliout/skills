## 附录一：名称 + 一句话用途


### Skills 精简表

- `find-skills`：用于搜索、发现和查询社区技能。
- `frontend-design`：用于生成网页、Dashboard 和落地页界面。
- `web-artifacts-builder`：用于构建带路由和状态管理的复杂前端应用。
- `canvas-design`：用于生成架构图、流程图和技术示意图。
- `theme-factory`：用于统一文档、PPT 和页面的视觉主题。
- `vercel-react-best-practices`：用于检查和优化 React 最佳实践。
- `web-design-guidelines`：用于改进网页设计规范、排版和响应式体验。
- `vercel-composition-patterns`：用于优化前端组件组合、拆分和复用。
- `shadcn`：用于指导和生成符合规范的 shadcn/ui 组件代码。
- `vercel-react-native-skills`：用于指导 React Native 跨平台开发与优化。
- `technical-writer`：用于编写 README、API 文档和技术说明。
- `doc-coauthoring`：用于撰写技术方案、RFC 和长文档。
- `docx`：用于创建、编辑和转换 Word 文档。
- `pptx`：用于生成、编辑和转换 PPT 演示文稿。
- `pdf`：用于处理 PDF 合并、拆分、OCR 和表单填写。
- `xlsx`：用于清洗和分析 Excel/CSV 数据并生成图表。
- `planning-with-files`：用于把复杂任务拆解并持久化到进度文件中。
- `project-planner`：用于做项目规划、需求梳理和分阶段实施设计。
- `architecture-patterns`：用于推荐合适的系统架构模式和最佳实践。
- `architecture-decision-records`：用于生成 ADR 架构决策记录。
- `requesting-code-review`：用于做更专业的代码审查和质量检查。
- `memory-intake`：用于把经验、规范和踩坑记录写入记忆库。
- `memory-audit`：用于检查记忆库中过时或冲突的内容。
- `memory-evolution`：用于整理、精简和优化记忆库结构。
- `webapp-testing`：用于生成和执行基于 Playwright 的 E2E 测试。
- `test-driven-development`：用于按 TDD 方式先写测试再写实现。
- `brainstorming`：用于围绕问题快速发散思路和方案。
- `systematic-debugging`：用于按结构化流程排查 bug 和定位根因。
- `writing-plans`：用于拆解需求并生成详细开发计划。
- `executing-plans`：用于按既定计划推进开发和跟踪进度。
- `audit-website`：用于扫描网站常见安全漏洞并输出审计建议。
- `skill-creator`：用于创建和封装自己的自定义技能。

### MCP 精简表

- `neural-memory`：用于提供跨会话、跨项目的长期结构化记忆能力。
- `playwright`：用于浏览器自动化控制、E2E 测试和页面验证。
- `filesystem`：用于访问和操作本地工作区文件系统。
- `sequential-thinking`：用于增强复杂问题的多步推理能力。
- `web_reader`：用于抓取公开网页内容并转换为 Markdown。
- `figma-developer-mcp`：用于读取 Figma 设计稿结构、样式和资源。
- `supercharged-figma`：用于实时编辑 Figma 画布和批量操作设计节点。
- `4_5v_mcp`：用于分析图片内容并识别 UI 组件与场景信息。
