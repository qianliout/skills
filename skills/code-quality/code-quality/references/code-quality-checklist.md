# Code Quality Checklist

在代码审查涉及错误处理、性能和边界条件时使用。

## Error Handling

- 是否吞掉错误，只记录日志却继续成功返回。
- 是否缺少关键 I/O、网络、解析、序列化、并发操作的错误处理。
- 是否把内部细节暴露给终端用户。
- 是否缺少 fallback、重试边界、告警或调试上下文。

## Performance and Resource Usage

- 是否存在 N+1 查询、重复计算、热路径高成本操作。
- 是否把整批大对象一次性加载进内存，缺少分页、流式处理或批量化。
- 是否缓存了昂贵结果；如果缓存了，是否有 TTL、失效策略和隔离边界。
- 是否存在集合无上限增长、循环内重复分配、频繁序列化或阻塞式 I/O。

## Boundary Conditions

- `nil`、空字符串、空切片、空 map、零值是否被正确处理。
- 数值上下界、分页边界、索引、计数、除零和 off-by-one 是否安全。
- 用户输入为超长、空白、异常编码或特殊字符时会怎样。
- 失败分支和回滚分支是否真正可达并被验证。

## Silent Failure Risks

- 失败是否会被误判为成功。
- 重试是否会制造重复副作用。
- 部分成功时，状态是否可能残缺或不一致。
- 是否存在只有在生产流量下才暴露的问题，例如缓存污染、批量数据、超时链路。

## Review Questions

- 这个操作失败时，调用方能否明确感知。
- 数据规模扩大 10 倍到 100 倍时，这段逻辑是否还能工作。
- 这里的零值、空值和默认值是否有歧义。
- 是否有最小测试能覆盖最危险的失败路径。
