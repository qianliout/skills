# Removal Plan

在评估删除候选、废弃能力下线或清理冗余实现时使用。

## Priority

- `P0`: 必须立即删除，通常因为安全风险、明显成本或阻断后续工作。
- `P1`: 应在当前迭代删除。
- `P2`: 放入后续迭代并给出迁移计划。

## Safe To Remove Now

- 位置：给出明确文件或符号范围。
- 依据：无引用、历史兼容层已失效、功能开关长期关闭、被新实现完全替代。
- 影响：说明删除后谁会受影响，为什么可以现在删。
- 验证：列出最小测试、运行方式或监控点。

## Defer With Plan

- 延后原因：仍有调用方、需要数据迁移、需要产品或运维窗口。
- 前置条件：例如指标连续归零、开关关闭一段时间、替代路径已验证。
- 迁移步骤：按阶段说明替代、通知、灰度、删除和回滚。
- 验证与回滚：说明如何确认删除安全，出问题后如何恢复。

## Checklist

- 已搜索静态引用和可能的动态使用路径。
- 已确认外部 API、脚本、文档或配置没有隐藏依赖。
- 已同步测试、文档和运维说明。
- 已明确 owner、时间点和风险控制方式。
