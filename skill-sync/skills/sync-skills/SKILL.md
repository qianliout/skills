# Sync Skills

把本地 Codex skill 目录作为唯一事实来源，并同步到其他编程工具的规则或技能目录。

## Workflow

1. 确认源目录。默认使用当前工作目录；源目录可以是单个 skill，也可以是包含多个 skill 子目录的根目录。
2. 预览同步：

```bash
./sync-skills --dry-run
```

3. 执行同步：

```bash
./sync-skills
```

4. 如目标项目不是当前目录，显式传入项目根目录：

```bash
python3 skill-sync/scripts/sync_skills.py /path/to/skills --project /path/to/project --targets cursor,trae
```

## Default Targets

- `codex`: 复制完整 skill 目录到 `~/.codex/skills/<skill-name>/`。
- `cursor`: 生成项目规则到 `<project>/.cursor/rules/<skill-name>.mdc`。
- `trae`: 生成项目技能到 `<project>/.trae/skills/<skill-name>.md`。

## Notes

- 使用 `--dry-run` 查看将要写入、更新或跳过的文件。
- 使用 `--target-root <name>=<path>` 覆盖任一目标目录，例如 `--target-root cursor=/tmp/project/.cursor/rules`。
- 使用 `--remove-stale` 删除由本工具生成但源目录已不存在的目标文件。
- 生成文件包含 `Generated by skill-sync` 标记；`--remove-stale` 只会删除带该标记的文件。
